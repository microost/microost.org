microost.org

"microost.org" is an Internet domain name for package names of open-sourced Java libraries.

When publishing a new small open-sourced Java library as a person, naming its Java package is a problem. Internet domain names are used for Java packages.

  • Everyone does not have their own personal domain name. Even if they have, using the personal domain name may not be the best solution as its lifetime may depend on the person.
  • Domain names of source code hosting services (such as github.com and bitbucket.org) are not always the best solution also due to their lifetimes.
  • Purchasing a new domain name for each library is not very reasonable.

"microost.org" intends to be a co-managed hosting of Java package namespace. It is managed by a person for the time being, though.

Q&A

How is a new package name registered?

Send a pull request to the repository.

What does "microost" mean?

"microost" is coined from "microroast" and "roost".

How is "microost" pronounced?

Like mike-roost.
